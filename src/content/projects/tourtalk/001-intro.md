## 프로젝트 개요

**TourTalk**은 지역 기반 여행 정보 제공과 학예사(큐레이터) 해설 상품을 연결하는 통합 여행 플랫폼입니다.

SSAFY 13기 1학기 관통 프로젝트로, 약 1개월간(2025.04.28 ~ 2025.05.28) 개발했습니다. 

## 개발 배경

여행을 준비할 때 관광지 정보, 예약, 후기, 해설 서비스가 각각 다른 플랫폼에 흩어져 있어 사용자 입장에서는 번거로움이 큽니다.

한편, 매년 학예사 자격을 취득하는 사람은 늘고 있지만 박물관·미술관 등 일부 기관을 제외하면 실무 활동 기회가 매우 제한적입니다.

TourTalk은 이 두 가지 문제를 함께 해결하고자 기획했습니다. 여행 정보와 해설 서비스를 한 곳에서 제공하고, 학예사에게는 자신의 해설 상품을 직접 등록하고 운영할 수 있는 채널을 제공합니다.

## 핵심 기능

### 관광지 정보 및 검색

인기 관광지, 테마별·지역별 관광지를 한눈에 확인할 수 있으며, 검색을 통해 원하는 여행지의 상세 정보를 조회할 수 있습니다. KakaoMap API를 활용해 지도 기반으로 위치를 확인할 수 있습니다.

### 여행 계획 관리

원하는 관광지를 추가해 여행 계획을 저장하고 불러올 수 있습니다. 직접 발굴한 장소는 핫플레이스로 등록해 다른 사용자와 공유할 수 있습니다.

### 해설 상품 예약 및 결제

학예사 회원은 해설 상품을 등록·수정·삭제하고 예약 현황을 관리할 수 있습니다. 일반 회원은 원하는 상품을 예약하고 PortOne 결제 API를 통해 결제하거나 취소할 수 있습니다.

### AI 기능

ChatGPT API를 활용해 두 가지 AI 기능을 제공합니다.

- **관광지 정보 요약** — 상세 정보 페이지에서 AI가 핵심 내용을 요약해 제공
- **챗봇** — 여행지 추천 및 여행 계획 조언

### 커뮤니티 및 관리자 페이지

사용자는 게시글과 댓글을 통해 여행 경험을 공유할 수 있습니다. 관리자는 별도 페이지에서 회원 정보와 커뮤니티 게시글을 관리합니다.

### 역할 기반 회원 관리

일반 회원과 학예사 회원으로 구분되며, 학예사는 자격 정보를 입력해 별도 가입합니다. 인증은 JWT 기반으로 처리하고, 클라이언트 상태는 Pinia로 관리합니다.

## 기술 스택 및 선택 이유

| 기술 | 선택 이유 |
|------|-----------|
| Vue.js | 컴포넌트 기반 SPA 개발, 학습 목적의 프레임워크 선택 |
| Java + Spring Boot | 안정적인 백엔드 구조와 Spring Security 연동 |
| Spring AI | Java에서 직접 OpenAI API를 호출하기 위한 라이브러리 |
| MySQL | 관계형 데이터 저장 |

## 아키텍처

프론트엔드(Vue.js)와 백엔드(Spring Boot) 두 레이어로 구성됩니다. AI 기능은 별도 서버 없이 Spring AI 라이브러리를 통해 Spring Boot에서 직접 OpenAI API를 호출합니다. 관광지 데이터는 한국관광공사 TourAPI로 수집해 DB에 사전 적재한 구조이며, 길찾기는 Kakao Mobility API를 실시간으로 호출합니다.

## 프로젝트 개요

**Speakle**은 빅데이터 기반 맞춤형 팝송 추천과 가사를 활용한 영어 학습을 결합한 플랫폼입니다.

SSAFY 13기 2학기 특화 프로젝트로, 약 5주간(2025.08.25 ~ 2025.09.29) 개발했으며, **광주 1반 1등 우수상**을 수상했습니다.

## 개발 배경

영어를 잘하고 싶어 시간을 투자하지만, 실제 회화 실력은 좀처럼 늘지 않는다는 걸 많은 학습자가 경험합니다.

그 이유를 생각해보면:

- 교재 중심의 딱딱하고 재미없는 학습
- 실생활에서 쓰지 않는 표현 위주의 커리큘럼
- 높은 학습 비용과 시간적 제약
- 동기 부족으로 인한 꾸준함의 어려움

아무리 좋은 학습 방법도 재미없으면 오래 가지 않습니다.

그래서 저희는 "즐겁게 듣고, 따라 부르고, 배우는 영어"라는 방향으로 Speakle을 기획했습니다. 평소 즐겨 듣는 팝송을 학습 콘텐츠로 연결함으로써, 재미와 학습 효과를 동시에 잡는 경험을 만들고자 했습니다.

## 핵심 기능

### 맞춤형 팝송 추천

원하는 **상황과 장소**를 입력하면, 빅데이터 추천 시스템이 그에 맞는 팝송을 추천합니다. Weaviate 벡터 DB와 약 96만 곡의 Spotify 가사 임베딩 데이터를 기반으로, 의미적으로 유사한 팝송을 찾아줍니다.

### 가사 기반 학습 콘텐츠

노래 상세 페이지에서 한국어 번역 가사와 함께 노래를 들을 수 있고, 가사에서 뽑아낸 **단어, 문장, 표현, 관용구**를 별도로 학습할 수 있습니다. 단순히 노래를 감상하는 것에서 나아가 실제 회화로 확장할 수 있도록 설계했습니다.

### 집중 학습 모드

팝송 가사를 활용한 세 가지 학습 방식을 제공합니다.

- **빈칸 채우기** — 가사의 일부 단어가 비워진 문제를 풀며 어휘를 학습
- **스피킹** — 가사를 직접 발음하고 녹음해 발음 정확도를 평가
- **딕테이션** — 노래를 들으며 가사를 직접 입력하는 받아쓰기

### 리워드 시스템

연속 학습 일수, 포인트 적립, 랭킹 기능으로 학습 지속 동기를 부여합니다. Spotify 계정 연동도 지원합니다.

## 기술 스택 및 선택 이유

| 기술 | 선택 이유 |
|------|-----------|
| React + TypeScript | 컴포넌트 기반 UI 개발과 타입 안전성 확보 |
| Java + Spring Boot | 안정적인 백엔드 구조와 풍부한 생태계 |
| Python + FastAPI | AI/ML 서비스를 빠르게 구현하기 위한 비동기 API 서버 |
| PostgreSQL | 관계형 데이터의 안정적인 저장 및 조회 |
| Weaviate | 팝송 가사 임베딩 벡터를 저장하고 의미 기반 유사도 검색을 수행하기 위한 벡터 DB |
| Redis | OAuth·인증코드 임시 저장, 분산 락, 이벤트 스트림(Streams/Pub/Sub), 번역 결과 캐싱, DLQ 등 다목적 활용 |
| Jenkins | GitLab 연동 CI/CD 파이프라인 자동화 |

## 아키텍처

![Speakle 아키텍처](/images/projects/speakle/architecture_speakle.png)

애플리케이션은 세 개의 레이어로 구성됩니다.

- **Frontend** — React 기반 SPA
- **Backend** — Spring Boot REST API 서버 (PostgreSQL, Redis 사용)
- **Data Service** — FastAPI 기반 AI 추천 서버 (Weaviate 벡터 DB 사용)

프론트엔드와 백엔드는 REST API로 통신하고, 팝송 추천 요청은 백엔드에서 Data Service로 위임해 처리합니다. Jenkins와 Docker를 통해 GitLab push 시 자동으로 빌드 및 배포가 이루어집니다.
